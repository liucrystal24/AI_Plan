## **16 周学习计划（每周 10–12 小时，偏前端）**

**第 0 周（0.5 周）环境就绪**

- 产出：一个模板仓库（Nuxt.js + TS）+ `.env` 配置 + 基础部署（任意平台都行）
- 目标：后面所有能力都在同一个项目里迭代，不到处做零散 demo

**第 1–2 周：LLM 对话 + 流式 UI（前端核心竞争力）**

- 产出：一个 Chat UI（支持 SSE 流式输出、停止生成、重试、复制、引用展示占位）
- 必做点：消息状态（pending/streaming/done/error）、断线恢复、超时提示

**第 3–4 周：结构化输出 + 工具调用（让“聊天”能办事）**

- 产出：在 UI 里展示“工具调用轨迹”（timeline）：模型 → 调用工具 → 返回结果 → 最终回答
- 工具示例（选 2 个就够）：网页搜索（或站内搜索）、计算器/日期、读取本地 mock 数据、调用你的业务 API
- 重点：前端把“模型正在做什么”可视化，用户信任感大幅提升

**第 5–7 周：RAG（知识库问答）+ 引用溯源（ToB 高频需求）**

- 产出：知识库问答页（或同页一个 Tab），回答必须带引用片段与来源链接
- 必做点：
    - 文档导入（先支持 Markdown/HTML，PDF 可后置）
    - Chunk 策略 + metadata（来源、更新时间、权限标签）
    - 检索结果 UI：命中的片段高亮、可展开、可反馈“有用/无用”

**第 8–9 周：质量闭环（评测 + 埋点 + 回放）**

- 产出：一个最小“对话回放/评测面板”（可以是开发者页面）
- 必做点：
    - 埋点：问题、检索命中文档、最终回答、耗时、token/成本（若拿得到）
    - 离线评测集：至少 50 条（覆盖：常问、边界、对抗、时效性）
    - 回归：改提示词/切分/检索后，能一键跑评测并对比指标

**第 10–12 周：业务 Agent（多步骤任务）+ 失败恢复**

- 产出：一个“任务型助手”（例：生成周报/会议纪要/工单归类/运营文案带校验清单）
- 必做点：
    - 多步骤可视化：步骤列表、当前进度、失败重试单步
    - 人在回路：用户可编辑中间结果再继续（这点很产品化）

**第 13–14 周：上线级能力（稳定、可观测、可控成本）**

- 产出：可部署版本 + 基础监控面板（哪怕是日志聚合也行）
- 必做点：限流、缓存（检索/embedding/答案）、敏感信息脱敏、错误告警策略

**第 15–16 周：作品集打磨 + 面试故事线**

- 产出：2 个可讲清楚价值的项目 + 1 份技术方案文档（PDF/Notion 都行）
- 面试故事线模板：问题 → 方案 → 评测方法 → 指标提升 → 成本/延迟 → 风险治理

---

## **每周怎么分配 10–12 小时（建议）**

- 6 小时：项目开发（必须产出代码/可演示功能）
- 3 小时：补概念（LLM/RAG/安全/评测）
- 1–2 小时：整理评测集 + 写复盘文档（直接变简历素材）

---

## **建议你做的 2 个作品集（偏前端且好面试）**

- 项目 1：**Copilot 风格知识库助手**
    - 亮点：流式 UI、引用溯源、检索片段可视化、反馈闭环、评测回归
- 项目 2：**任务型 Agent 助手（工具调用轨迹可视化）**
    - 亮点：timeline、步骤进度、单步重试、人在回路编辑、审计日志

---

## **下一步我建议你先定一个“主项目主题”**

你偏前端，我建议主题选一个你熟悉的行业场景（更容易做出真实数据与需求），比如：

- 面向研发：PR 总结/变更说明/技术方案生成
- 面向运营：活动复盘/数据解读/内容生成+校对清单
- 面向客服：知识库问答+工单自动归类

你回我一句：你更想做“研发/运营/客服”哪类助手？我就把上面 16 周计划进一步细化到“每周具体任务清单 + 你作品集该怎么写简历要点”。

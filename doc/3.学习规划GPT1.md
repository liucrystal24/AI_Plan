## 岗位

- **AI 应用工程师 / LLM 应用工程师**：把大模型接入业务（客服、知识库、搜索、办公助手、代码/内容生成），做 RAG、工具调用、工作流、评测与上线。
- **AI 前端工程师 / 智能交互工程师**：在前端做智能化交互（Copilot式体验、智能表单、语义搜索、实时对话），关注流式输出、延迟、可解释性、隐私与成本。
- **Agent/自动化工程师（偏后端）**：用工具调用 + 多步骤规划 + 状态管理，把“聊天”变成“办事”（报表生成、工单处理、运营自动化）。

## 岗位职责（以 LLM 应用/Agent 为例）

- 需求拆解：哪些环节适合用模型，哪些必须规则/检索/人审兜底
- 搭建链路：数据获取→切分→向量化→检索→重排→生成（RAG）
- 工具调用：让模型调用搜索、数据库、工单系统、内部API（含权限与审计）
- 质量与评测：命中率、事实性、引用率、拒答率、成本、延迟；做 A/B、回归集
- 上线与治理：缓存、限流、灰度、监控、提示词/检索配置版本化、合规与脱敏

## 技能栈

- **Python（必须）**：语法、包管理、虚拟环境、类型提示、基础工程结构；能写服务/脚本。
- **LLM 基础**：token、上下文窗口、温度/采样、函数/工具调用、结构化输出(JSON Schema)。
- **RAG 全链路**：文档清洗/切分、Embedding、向量库、检索策略、重排、引用与溯源。
- **工程化**：API 服务（FastAPI）、异步、队列任务、日志/指标/追踪、配置管理、Docker。
- **评测与提示词工程（工程化视角）**：离线评测集、自动评分+人工抽检、红队用例、回归。
- **基础 ML 常识（够用即可）**：过拟合/泛化、分类/回归、embedding含义、向量相似度。
- 加分项：SQL、Elasticsearch/检索、Linux、云服务、权限/合规、成本优化。

## 目标

能独立做一个可上线的“企业知识库问答/客服助手/运营助手”，带评测与监控。

## 16 周学习计划（默认每周 10–12 小时）

**第 1–2 周：Python + 基础后端能力**

- 目标产出：一个 FastAPI 小服务 + 单元测试 + 基本日志
- 内容：
    - Python 工程化（venv/poetry、lint/format、pytest）
    - FastAPI：路由、依赖注入、异常处理、流式响应（为对话流式输出做准备）
    - Docker 基础（会打包即可）

**第 3–4 周：LLM API 使用与对话工程**

- 目标产出：一个“流式对话 Web demo”（前端你很强，做出产品感）
- 内容：
    - Chat completion、系统提示词、结构化输出
    - 工具调用（function calling）：把“问答”变“办事”
    - 可靠性：超时/重试/降级、敏感信息过滤、提示词注入基本防护

**第 5–7 周：RAG 核心（决定你能不能做企业落地）**

- 目标产出：企业知识库问答（带引用、可追溯）
- 内容：
    - 文档处理：PDF/HTML/Markdown清洗、chunk策略、元数据（来源、更新时间、权限）
    - Embedding 与向量库（任选一个：FAISS/Chroma/pgvector 等）
    - 检索策略：top-k、混合检索（关键词+向量）、重排（cross-encoder 或 LLM rerank）
    - 生成策略：引用片段拼接、回答必须带来源、无法回答要拒答

**第 8–9 周：评测与质量闭环（很多人缺这块，你做了就拉开差距）**

- 目标产出：一套离线评测脚本 + 回归集
- 内容：
    - 构建评测集：真实问题、难例、对抗问题、时效性问题
    - 指标：正确率/引用率/拒答率/幻觉率、延迟与成本
    - 自动评测：规则 + LLM-as-judge（要有人工抽检与一致性策略）

**第 10–12 周：Agent 工作流（让你从“知识库”升级到“业务助手”）**

- 目标产出：一个能完成多步骤任务的助手（例如：生成周报→拉取数据→出图→写结论）
- 内容：
    - 工具编排：检索、数据库查询、内部HTTP API、文件生成
    - 状态管理：多轮上下文、任务进度、失败恢复
    - 权限与审计：不同用户可见数据不同、所有工具调用留痕

**第 13–14 周：上线能力（稳定、可观测、可控成本）**

- 目标产出：可部署版本（Docker Compose/单机部署）+ 基本监控
- 内容：
    - 缓存（embedding缓存、检索缓存、答案缓存）、限流、并发控制
    - 监控：请求量、延迟、错误率、token消耗、命中率
    - 灰度与回滚：提示词/检索配置版本化

**第 15–16 周：作品集打磨 + 面试准备**

- 目标产出：2 个“可讲清楚价值”的项目 + 1 份技术方案文档
- 内容：
    - 项目复盘：问题→方案→评测→结果→成本→风险与治理
    - 高频面试点：RAG失败案例、如何降低幻觉、如何做权限、如何评测、如何控成本
